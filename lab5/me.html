Preguntas de la actividad

¿Cuál es la diferencia entre Internet y la World Wide Web?

El internet es el medio, lo que implica a todas las computadoras contectadas entre sí. En cambio la World Wide Web son todas las páginas que existen dentro de esas computadoras para ser accedidas por los usuarios.

¿Cuáles son las partes de un URL?

Una URL se divide en varias secciones, entre ellas el Protocolo (como http://), el subdominio (ej. 'www'), dominio (ej. 'google.com'), ruta (ej. /drive/u/0/),y finalmente el nombre y tipo del archivo (ej. '/index.html').

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: Obtener información de un servidor en modo de lectura de datos

  • HEAD: Similar a GET, obtiene información, pero sin arrojar una respuesta

  • POST: Envía información, generando un cambio en el estado o datos del servidor

  • PUT: Envía información que reemplaza todos los datos en el destino por el valor enviado

  • PATCH: Aplica ciertas modificaciones al recurso obtenido

  • DELTE: Borra un dato/recurso del servidor

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se debe utilizar un método POST, para que no se pueda modificar y solo se compare con las credenciales registradas para ingresar.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, que reciba la información sobre ese sitio web y el mavegador pueda desplegarlo en pantalla.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la operación se realizó sin problemas, ya sea el envío o recibo de datos.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, porque signfica que la página no ha sido encontrada, por lo que el desarrollador debe comprobar la conexión y accesibilidad a la misma.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, porque el error 500 ocurre cuando se recibe una respuesta inesperada del servidor. Por lo que se tendría que comprobar si existe alguna configuración deficiente en el acceso al servidor, permisos, etc.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que ese atributo ya no está disponible en la versión de HTML5, y que se reemplaza por alguno otro (por ejemplo con el uso de CS5). Ejemplos comunes son: "< image>:name", "< link>:target", "< html>:version".

¿Cuáles son las diferencias principales entre HTML4 y HTML5?

  • COMPATIBILIDAD:En las versiones más recientes de los navegadores, las páginas HTML4 puede que no se muestren correctamente

  • MULTIMEDIA: Anteriormente en HTML4 se requería de usar Flash o Javascript para reproducir animaciones y ahora en HTML5, se puede hacer meramente con CSS

  • ETIQUETAS: Anteriormente era común utilizar "< div id="etiqueta">" y ahora ya existen etiquetas "< header>" "< nav>", "< article>"

¿Qué componentes de estructura y estilo tiene una tabla?

  • < table>: Define una tabla

  • < th>: Define los encabezados de una tabla

  • < tr>: Define una fila de una tabla

  • < colgroup>: Selecciona un conjunto de columnas de la tabla

¿Cuáles son los principales controles de una forma HTML5?

Según la página developer.mozilla.org, los controles básicos de forma HTML5 son:
< form>, < fieldset>, < legend>, < textarea>, < label>, < button> & < input>
